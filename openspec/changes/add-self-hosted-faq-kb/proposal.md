# 变更：添加自建 FAQ 知识库系统

## 原因
当前的 FAQ 系统使用静态 mock 数据，缺乏与真实知识库的集成。用户需要一个动态的、可管理的 FAQ 系统，支持内容管理、搜索和多语言功能。之前尝试与 Zammad 知识库集成因 API 令牌权限问题而失败，因此我们需要实现一个自建解决方案。

## 变更内容
- 用自建的数据库支持的知识库替换 mock FAQ 数据
- 为 FAQ 分类和文章实现 CRUD 操作
- 添加搜索功能和关键词匹配
- 支持多语言内容管理
- 创建用于 FAQ 内容管理的管理界面
- 实现缓存以提高性能

## 影响
- 受影响的规范：faq-system
- 受影响的代码：
  - src/app/api/faq/ 中的 API 路由
  - src/lib/mock-faq-data.ts 中的数据层
  - 管理面板组件
  - 面向客户的 FAQ 组件
- 破坏性变更：用真实的数据库操作替换 mock 数据实现
